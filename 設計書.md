# reversi（仮称：黒白リバーシ）

---

## 📌 要求定義（What）

### 1. サービス名

- 黒白リバーシ（仮称）
    

### 2. 想定ユーザー

- スマートフォンまたはPCで、同一端末上でリバーシを2人対戦プレイしたいユーザー
    
- 途中で中断しても「続きから」遊べるようにしたいユーザー
    

### 3. ユースケース概要

- ユーザーは「新規ゲーム」または「続きから再開」を選択してゲームを開始
    
- ターンごとに交互に石を置き、ルールに従って相手の石をひっくり返す
    
- 有効な手がない場合は自動パス
    
- ゲーム終了時にはアニメーション付きで石をカウントし、勝敗を表示
    
- ゲーム状態はCookieに保存され、次回アクセス時に再開可能

## ✅ 要件定義（How）

---

### 1. 基本仕様

|項目|内容|
|---|---|
|利用環境|スマートフォン・PCブラウザ（レスポンシブ対応）|
|表示方式|スマホ：全画面／PC：中央配置（固定幅）|
|フレームワーク|Vue 3（Composition API）、Tailwind CSS|
|状態管理|Vueのローカルステート＋Cookie|
|データ保存|Cookie（ゲーム状態）|
|バックエンド|不要（フロントエンドのみ）|

---

### 2. UI構成（画面設計）

📱 メイン画面

```
[ 黒の手番 ● ]  or  [ 白の手番 ○ ]    ← 上部インジケータ

[ 8×8 盤面（石配置）]

[ 新規ゲーム ]   [ 続きから再開（条件付き） ]

[ ゲーム終了時：結果表示モーダル ]
→ [ 黒：XX枚 ● ] [ 白：YY枚 ○ ]
→ [ 勝者：● or ○ or 引き分け ]
```

🎮 操作要素

|要素|内容|
|---|---|
|マス|クリック／タップで石を置く|
|石のアニメーション|置く時、ひっくり返す時にCSSアニメーション|
|自動パス|有効手なしの場合、相手手番に自動切替|
|ゲーム終了|双方置けなくなったら自動終了、石数カウントアニメーション|
|結果表示|勝者表示＋再試合誘導ボタン表示|

---

### 3. ルール実装

|項目|内容|
|---|---|
|盤面構成|8×8の2次元配列|
|初期配置|中央4マスに白黒2枚ずつ配置|
|ターン制御|黒（●）先手、交互に交代（自動）|
|合法手判定|置いた位置から8方向に相手色を挟むか確認|
|自動パス|合法手が存在しない場合、自動的にスキップ|
|ゲーム終了|両者とも置けなくなった時点で終了|
|勝敗判定|石の数で多い方が勝利。引き分けあり|

---

### 4. アニメーション仕様

|項目|内容|
|---|---|
|石の設置|`scale(0)` → `scale(1)` のポップアニメーション|
|ひっくり返し|`rotateY(180deg)`の反転アニメーション|
|カウントアップ|石数を1つずつ表示、最終結果をモーダルで提示|
|CSS管理|Tailwind＋`@keyframes`定義で実装|

---

### 5. ゲーム状態保存（Cookie）

|タイミング|保存・復元|
|---|---|
|保存内容|`board`, `turn`, `gameOver`, `timestamp`|
|保存形式|JSON文字列をCookie（`reversi_state`）に保存|
|保存タイミング|石設置完了後、ターン交代後、自動パス後|
|復元条件|Cookieが有効であり、構造が正しい場合のみ「続きから再開」ボタン表示|

```json
{
  "board": [[0,0,0,...],[...],...],
  "turn": 1,
  "gameOver": false,
  "timestamp": 1720000000000
}
```

---

### 6. 非機能要件

|項目|内容|
|---|---|
|レスポンシブ対応|画面幅に応じて盤サイズ・UI自動調整|
|操作レスポンス|100ms以内でUI反応|
|モバイル最適化|タップ範囲拡大・スクロール防止|
|Cookie有効期限|最終操作から30日間有効|
|セキュリティ|Cookieに個人情報なし、SameSite=Lax|

---

### 7. コンポーネント構成（Vue 3）

|コンポーネント名|役割|
|---|---|
|`App.vue`|全体構成・テーマ・状態管理|
|`Board.vue`|盤面描画・クリックイベント処理|
|`Stone.vue`|石の表示とアニメーション|
|`ControlPanel.vue`|「新規ゲーム」「続きから再開」ボタン|
|`ResultModal.vue`|終了時の石数集計＆結果表示|
|`CookieUtil.ts`|Cookieの保存・読み出しユーティリティ|
|`ReversiLogic.ts`|合法手判定・石ひっくり返し・パス処理|

---

### 8. スマートフォン最適化仕様

|機能|内容|
|---|---|
|フルスクリーン表示|`height: 100vh`, `overflow: hidden`|
|スクロール防止|`touch-action: none`|
|盤サイズ自動調整|`vw`ベースでマス幅を計算|
|SafeArea対応|`env(safe-area-inset-*)`をpaddingに使用|

---

### 9. PWA・OGP対応（任意）

|OGP画像|`/ogp-reversi.jpg`（1200×630）|
